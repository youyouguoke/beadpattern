0. SEO增长的本质（先统一认知）

你的产品不是：

❌ bead pattern tool
❌ CMS

而是：

✅ “Google 长尾流量收割系统”

SEO增长公式（核心）
Traffic =
Pages × Indexability × Internal Link Strength × Keyword Coverage × CTR
1. 🧱 SEO页面结构（决定上限）
1.1 页面类型矩阵（必须固定）
1. /pattern/{slug}      → 核心流量页（80%）
2. /tag/{slug}          → 流量入口页（15%）
3. /difficulty/{level}  → 辅助流量页（3%）
4. /trending            → 热点页（2%）
2. 🔑 长尾关键词体系（核心增长引擎）
2.1 Keyword结构（必须标准化）
[adjective] + [object] + bead pattern
示例：
cute panda bead pattern
easy cat bead pattern
pixel heart bead pattern
kawaii animal bead pattern
2.2 Keyword生成规则（CMS内建）

每个 Pattern 必须自动生成：

{
  "primary": "cute panda bead pattern",
  "secondary": [
    "easy panda bead pattern",
    "kawaii panda beads",
    "pixel panda craft pattern"
  ]
}
3. 🧭 Tag SEO系统（最重要增长入口）
3.1 Tag不是分类，是“SEO landing page”

每个 tag = 一个独立 Google 页面

3.2 Tag页面结构（必须统一）
H1: Cute Animal Bead Patterns

Intro paragraph (SEO text)

Pattern grid (20–50 items)

Internal links

FAQ section
3.3 Tag类型分层
Tier 1: animal, kawaii, pixel
Tier 2: panda, cat, dog
Tier 3: cute panda, chibi cat
🚨 关键策略（非常重要）

❗不要只做“标签列表”，要做“SEO文章化标签页”

4. 🔗 Internal Linking（SEO核心放大器）
4.1 链接规则（必须执行）

每个 pattern 页面必须包含：

同 tag 相关 pattern（5–10个）
同 difficulty pattern
trending patterns
4.2 链接结构（权重传递）
Pattern → Tag
Pattern → Pattern
Tag → Pattern
Home → Tag
4.3 权重策略
Homepage → 100
Tag page → 80
Pattern page → 60
5. 🧠 内容密度策略（决定能不能进前3页）
5.1 Pattern页面必须包含：
≥300 words description
steps explanation
usage context
FAQ
5.2 SEO文本结构模板
What is this pattern?
How to use it?
Difficulty explanation
Materials needed
Related patterns
FAQ
6. 🚀 Sitemap策略（Google抓取核心）
6.1 sitemap结构
sitemap.xml
 ├── /pattern-sitemap.xml
 ├── /tag-sitemap.xml
 ├── /trending-sitemap.xml
6.2 更新机制
pattern publish → instant update
tag update → daily batch
trending → hourly cache
7. 🔥 Indexing策略（决定收录速度）
7.1 三阶段策略
Stage 1（启动）

internal linking boost
Stage 2（增长）
update frequency increase
fresh content push
Stage 3（规模）
auto ping sitemap
